COMPUBUILDER:

PC BUILDING APPLICATION

CompuBuilder is an application that provides a user with a PC build parts list based on their budget and intended use. My second full-stack web application project, I created this project for my senior project at Arizona State University.

Users are asked for their name, their budget, and what they will be using their PC for. Their progress is tracked by the progress bar, and they can hit the back button to make modifications at any time.

Users can then view the generated list of PC parts for their build. The logic in the application ensures the parts are compatible. For example, Motherboard and CPU socket types must match, and power supplies must supply enough wattage for the cpu and gpu.

By selecting more details, users can see the specifications of each of the components in their build list. This way, more experienced users can verify the component will work the way they need for their build.

The project is based on MERN, a technology stack that utilizes MongoDB document database, the Express.js web framework, React.js client-side framework, and the Node.js server framework.

The front-end client side utilizes API calls that communicate with the server, which contains the API routes, data models, and controllers that retrieve information from the database.

This project was meant as a proof of concept for a streamlined PC-build planning application. It’s meant to make the PC build planning experience less difficult, intimidating, and time consuming for both new and experienced PC builders.
